# AstroVerse Animation & Transition Specifications

> Generated by UX Designer 2 — CHI-60
> Date: 2026-04-04

---

## 1. Animation Philosophy

- **Purposeful motion** — every animation communicates state change or draws attention
- **Consistent timing** — same durations for same interaction types across the app
- **Respectful of preferences** — all animations disabled when `prefers-reduced-motion: reduce`
- **60fps target** — only animate `transform` and `opacity`; avoid layout-triggering properties

---

## 2. Timing Scale

| Token     | Duration | Easing                         | Usage                           |
|-----------|----------|--------------------------------|---------------------------------|
| instant   | 100ms    | ease-out                       | Micro-feedback (press states)   |
| fast      | 150ms    | ease [0.4, 0, 0.2, 1]        | Hover, focus, toggle            |
| normal    | 300ms    | ease [0.4, 0, 0.2, 1]        | Standard transitions, reveals   |
| slow      | 500ms    | ease [0.4, 0, 0.2, 1]        | Page transitions, complex anims |
| entrance  | 400ms    | ease-out                       | Element entrances               |
| exit      | 200ms    | ease-in                        | Element exits                   |

### Spring Presets

| Preset  | Stiffness | Damping | Usage                         |
|---------|-----------|---------|-------------------------------|
| smooth  | 300       | 30      | Cards, panels, general        |
| bouncy  | 400       | 20      | Interactive elements, badges  |
| gentle  | 200       | 40      | Background elements, ambient  |

## 3. Component Animations

### Button

```
Hover:   scale(1.02), 150ms smooth spring
Active:  scale(0.98), 100ms instant
Loading: spinner rotates, text fades to 60% opacity
Focus:   glow ring appears, 150ms ease
```

### Card

```
Hover:   translateY(-2px), shadow-glow, 300ms ease
Active:  translateY(0), 100ms instant
Glass hover: background opacity 0.05 → 0.10, 300ms ease
Enter:   fadeInUp, 400ms ease-out, 0.05s stagger in grids
```

### Modal

```
Enter:   backdrop fadeIn 150ms → content scaleIn (0.95→1) + fadeIn, 300ms spring
Exit:    content scaleOut (1→0.95) + fadeOut, 200ms ease-in → backdrop fadeOut 150ms
```

### Tabs

```
Indicator: translateX with spring (stiffness: 300, damping: 30)
Content:   fadeIn, 150ms ease (current exits same time as new enters)
```

### Dropdown

```
Enter:   scaleY (0.95→1) + fadeIn, 200ms spring, transformOrigin: top
Exit:    scaleY (1→0.95) + fadeOut, 150ms ease-in
```

### Tooltip

```
Enter:   scale (0.96→1) + opacity (0→1), 150ms ease-out, 200ms delay
Exit:    opacity (1→0), 100ms ease-in
```

### Toast

```
Enter:   slideIn from right + fadeIn, 300ms spring
Exit:    slideOut to right + fadeOut, 200ms ease-in
Stack:   items translateY when one exits, 200ms spring
```

### Toggle

```
Knob:    translateX, 200ms spring (stiffness: 400, damping: 25)
Track:   background-color transition, 200ms ease
```

## 4. Page Transitions

### Route Change

```
Exit: current page opacity 1→0 + translateY(0→-10px), 150ms ease-in
Enter: new page opacity 0→1 + translateY(10px→0), 300ms ease-out
Overlap: none (exit completes before enter starts)
```

**Implementation:** React Router + Framer Motion `AnimatePresence`

### Tab Content Switch

```
Exit: opacity 1→0, 100ms ease-in
Enter: opacity 0→1, 150ms ease-out
No transform (content swap only)
```

## 7. Interaction Feedback

### Drag & Drop

```
Pickup: scale(1.05), shadow-glow, 200ms spring
Drag:   slight rotate(2deg), opacity 0.9
Drop:   scale(1), rotate(0), 300ms spring (bouncy)
        ripple effect at drop point, 300ms ease-out
```

### Form Validation

```
Error shake: translateX(0→-5→5→-5→5→0), 300ms ease
Success check: scaleIn + checkmark draw, 300ms
```

### Loading States

```
Spinner: rotate(0→360deg), 1s linear infinite
Dots:    opacity cycle (0→1→0), stagger 0.2s, 1.2s infinite
Progress bar: width transition, 500ms ease-out per update
```

### Scroll Reveal

```
Trigger: IntersectionObserver with 0.1 threshold
Animation: fadeInUp — opacity 0→1, translateY 20px→0
Duration: 400ms ease-out
Stagger: 0.05s between sibling elements
```

## 8. Reduced Motion

When `prefers-reduced-motion: reduce` is active:

| Animation         | Fallback                              |
|-------------------|---------------------------------------|
| All transforms    | Instant state change (0ms)            |
| All opacity       | Instant toggle (0ms)                  |
| Chart wheel draw  | All elements visible immediately      |
| Page transitions  | Instant swap, no animation            |
| Shimmer/skeleton  | Static gray placeholder               |
| Hover effects     | Color change only (no scale/translate)|
| Modals            | Instant show/hide                     |
| Spring physics    | All replaced with 0ms instant         |

**Implementation:**

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

And in Framer Motion:

```typescript
const reducedMotion = useReducedMotion();
// Pass reducedMotion ? { duration: 0 } : normalVariant to all animations
```

## 9. Performance Budget

| Metric                  | Target          |
|-------------------------|-----------------|
| First contentful paint  | < 1.5s          |
| Animation frame budget  | < 16.67ms/frame |
| JS animation payload    | < 50KB gzipped  |
| Concurrent animations   | Max 3 at once   |
| Layout shifts (CLS)     | < 0.1           |

**Rules:**
- Only animate `transform` and `opacity`
- Use `will-change` sparingly (only on elements actively animating)
- Avoid animating `box-shadow` on mobile (use `opacity` on pseudo-elements instead)
- Debounce scroll-triggered animations
- Use `requestAnimationFrame` for JS-driven animations
